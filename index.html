utc-ds

A terminal-inspired design system for developer portfolios and project websites. Built with pure CSS, zero JavaScript dependencies, and a dark-first philosophy.

Inspired by PlanetScale's website aesthetic and charmbracelet/glow's terminal markdown rendering. Change --ds-primary and --ds-secondary to re-theme everything.

Utkarsh Chourasia | @utc | 2026 | Design System

Inline Semantics

Here's a hyperlink styled with the accent color. Use strong for importance, emphasis for stress, inline code for technical terms, ⌘ + K for keyboard shortcuts, highlighted text for attention, deleted text for corrections, and DS for abbreviations. We also support superscripts and subscripts.

Code

Code Blocks

Inline code uses JetBrains Mono with a subtle background. Block code gets a bordered container:

#!/bin/bash
# Deploy script for production
set -euo pipefail

echo "Building Docker image..."
docker build -t myapp:latest .

echo "Pushing to registry..."
docker push registry.example.com/myapp:latest

echo "Deploying to cluster..."
kubectl rollout restart deployment/myapp

echo "✓ Deployment complete"

And here's a configuration example:

# docker-compose.yml
version: '3.8'
services:
  app:
    build: .
    ports:
      - "8080:80"
    environment:
      DATABASE_URL: postgres://db:5432/myapp
    depends_on:
      - db
  db:
    image: postgres:16
    volumes:
      - pgdata:/var/lib/postgresql/data

volumes:
  pgdata:

Note

This design system uses color-mix() in oklch for derived colors. This is a Baseline 2023 feature supported in all modern browsers.

Tip

Change only --ds-primary and --ds-secondary in tokens.css to re-theme the entire system.

Warning

Using @import in production may cause render-blocking cascaded requests. Concatenate or bundle for production use.

Caution

Overriding base element styles with framework-specific resets may cause conflicts. Load utc-ds.css after your framework's reset.
